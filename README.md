# 📚 Book Store Sales Analysis – SQL Project

### 📌 Project Overview

This project is a Book Store Sales Analysis System developed using SQL to manage and analyze bookstore operations. The database stores information about books, customers, and orders while performing various business analysis queries to generate meaningful insights.

The project demonstrates the use of:

- Database creation
- Table relationships
- Data analysis using SQL queries
- Aggregate functions
- Joins
- Filtering and sorting techniques

### 🎯 Objectives
- Manage bookstore inventory and customer records.
- Track customer orders and sales transactions.
- Analyze sales performance and revenue generation.
- Identify top-selling books and popular genres.
- Monitor stock availability after orders.
- Perform customer and order analysis using SQL queries.

### 🛠️ Technologies Used
- SQL
- PostgreSQL
- CSV Import
- DBMS Concepts

### 🗂️ Database Structure

#### 📘 Books Table

Stores information about books:

- Book ID
- Title
- Author
- Genre
- Published Year
- Price
- Stock

#### 👥 Customers Table

Stores customer details:

- Customer ID
- Name
- Email
- Phone
- City
- Country

#### 🛒 Orders Table

Stores order transaction details:

- Order ID
- Customer ID
- Book ID
- Order Date
- Quantity
- Total Amount

### 🔍 SQL Operations Performed

#### ✅ Basic Queries
- Retrieve books by genre
- Find books published after 1950
- List customers from specific countries
- Display orders from a specific month
- Calculate total stock available
- Find most expensive books
- Identify orders with quantity greater than 1
- Retrieve high-value orders
- List all available genres
- Find books with lowest stock
- Calculate total revenue

#### ✅ Advanced Queries
- Total books sold by genre
- Average price by genre
- Customers with multiple orders
- Most frequently ordered book
- Top expensive books by genre
- Total quantity sold by author
- Cities with high-spending customers
- Customer who spent the most
- Remaining stock after fulfilling orders

### 📊 Key Insights
- Helps identify top-performing books and genres.
- Tracks overall bookstore revenue and customer spending patterns.
- Provides inventory management insights through stock analysis.
- Enables customer behavior analysis using order history.
- Demonstrates practical use of SQL joins, aggregations, and filtering.

### 🚀 How to Run the Project
- Create a PostgreSQL database.
- Run the table creation SQL scripts.
- Import CSV data into tables.
- Execute SQL queries for analysis.
- View insights generated from the database.

### 📌 Conclusion

This SQL project provides a complete bookstore sales and inventory analysis solution using relational database concepts. It demonstrates how SQL can be used to manage business data efficiently and generate valuable analytical insights for decision-making.
